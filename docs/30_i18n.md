# i18n strategy

Do NOT store multiple language columns in domain tables.
Domain tables store translation keys, e.g.:
- item.title_key = "item.battery.title"
- disposal_method.name_key = "disposal.restmuell.name"

Translations live in:
- core.i18n_translation(key, lang, text)

Fallback:
- requested lang -> "de" -> "en" -> key as last resort

Current import behavior:
- Imports currently populate only `de`; other languages are added later via translation pipeline/admin.
